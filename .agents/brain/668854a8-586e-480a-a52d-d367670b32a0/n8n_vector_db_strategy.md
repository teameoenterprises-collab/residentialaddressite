## 🛠️ Technical Architecture (The n8n Blueprint)

### 1. The Trigger & Extraction (Low Cost)
- **Schedule**: Every 4 hours (balance between latency and API limits).
- **Reddit Node**: "Get Many Posts" (Limit: 100, New Posts).
- **Set Node (Data Cleaning)**: Only keep: `Post ID`, `Title`, `Content` (sliced to 4000 chars), `Subreddit`, `Permallink`, `Created UTC`.
- **Date/Time Filter**: Immediately discard posts older than the 4-hour window.

### 2. The Cost-Saving Filter (AI Pre-Filter)
- **Layer 1**: Gemini 1.5 Flash (Cheap).
- **Logic**: "Does this post have anything to do with [Insurance/LLCs/Banking]?"
- **Impact**: Removes 70-90% of junk data, saving hundreds in AI credits for the Pro level evaluation.

### 3. The Accuracy Engine (Vector DB + Pro AI)
- **Vector DB**: Superbase (compatible with n8n Vector Store).
- **Few-Shot Logic**:
    1. Post comes in.
    2. System searches Superbase for the **5 most similar past examples** (Human-labeled).
    3. Gemini 2. Pro receives: [Current Post] + [5 Relevant Examples with Results].
- **Accuracy**: Jumps from 80% to 99% because the AI sees exactly how a human handled similar specific threads.

### 4. The Human-in-the-Loop Loop
- **Airtable Sync**: Sync `Examples` table (Manually verified) to Superbase Vector DB once every 24h.
- **Feedback Mechanism**: If MarkBTM finds an AI mistake (error in judgment), he moves that post to the `Examples` table with the correct label. The system "learns" instantly for next time.
